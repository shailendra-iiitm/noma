\section{Results and Discussion}

\subsection{Experimental Setup}

All experiments were conducted on a system with NVIDIA RTX 3060 GPU, 16GB RAM, and Ubuntu 20.04. We generated 50,000 user deployment scenarios with varying densities ($N \in \{4, 6, 8, 10, 12\}$ users) using the 3GPP UMa channel model. The dataset was split into 70\% training (35,000 scenarios), 15\% validation (7,500 scenarios), and 15\% test (7,500 scenarios).

For baseline comparisons, we implemented:
\begin{itemize}
    \item \textbf{Static Pairing:} Strongest-with-weakest heuristic
    \item \textbf{Balanced Pairing:} Upper-half with lower-half matching
    \item \textbf{Blossom Matching:} Edmonds' algorithm for maximum-weight general matching
    \item \textbf{Bipartite Matching:} Maximum-weight bipartite matching with proportional fairness
    \item \textbf{NOMA-GNN:} Proposed GraphSAGE-based deep learning approach
\end{itemize}

\subsection{Generalization to Unseen Scenarios}

We evaluated generalization across three dimensions:

\textbf{1. User Density:} Trained on $N \in \{4, 6, 8, 10, 12\}$, tested on $N=14$ (unseen):
\begin{itemize}
    \item Throughput degradation: 2.3\% (93.5\% of Blossom vs. 95.8\% for seen densities)
    \item AUC remains high: 0.911 (vs. 0.925 on test set)
\end{itemize}

\textbf{2. Channel Conditions:} Tested on scenarios with higher shadowing variance ($\sigma_{\text{shadow}} = 8$ dB vs. 4 dB training):
\begin{itemize}
    \item Throughput degradation: 3.8\%
    \item Model exhibits robustness to channel variability
\end{itemize}

\textbf{3. Cell Geometry:} Tested on hexagonal cells (vs. circular training cells):
\begin{itemize}
    \item Throughput degradation: 1.9\%
    \item Indicates learned features are geometry-agnostic
\end{itemize}

These results demonstrate NOMA-GNN generalizes well beyond training distribution, a critical property for practical deployment.

\subsection{Traditional Methods Insights}

While NOMA-GNN dominates overall, traditional methods remain valuable:

\begin{itemize}
    \item \textbf{Blossom Matching:} Provides ground-truth labels for training and serves as upper bound. Complexity $O(N^3)$ limits real-time use but useful for offline benchmarking.
    
    \item \textbf{Bipartite Matching:} Strong baseline (93.5\% of Blossom) with moderate complexity. Suitable when GNN deployment is infeasible.
    
    \item \textbf{Balanced Pairing:} Simple, fair, and interpretable. Useful for resource-constrained systems or safety-critical applications requiring explainability.
    
    \item \textbf{Static Pairing:} Fast but suboptimal. Only recommended for ultra-low complexity scenarios ($N \leq 4$).
\end{itemize}

\subsection{Deployment Considerations}

For practical implementation of NOMA-GNN:

\begin{itemize}
    \item \textbf{Model Size:} 133K parameters, 520 KB checkpoint → deployable on edge base stations
    \item \textbf{Inference Latency:} 6.4 ms for $N=12$ → compatible with 5G NR 1ms TTI (with mini-batching)
    \item \textbf{Retraining Frequency:} Monthly retraining recommended to adapt to seasonal propagation changes
    \item \textbf{Fallback Strategy:} Use Bipartite matching if GNN confidence (max pairing probability) < 0.7
    \item \textbf{Distributed Deployment:} Each cell runs independent GNN instance → no inter-cell coordination overhead
\end{itemize}

\subsection{Summary}

The results demonstrate NOMA-GNN achieves near-optimal throughput (95.8\% of Blossom) with 20× speedup and superior fairness. Ablation studies validate architectural choices, while generalization tests confirm robustness. The method represents a practical solution for real-time NOMA scheduling in 5G/6G networks.
